# コンフリクト練習

コンフリクトは、複数人が同じファイルの同じ場所を変更したときに起きやすいです。

## コンフリクトが起きたときの基本

あわてて全部消さず、まずファイルの中を確認します。

```text
<<<<<<< HEAD
自分のブランチの変更内容
=======
mainブランチ側の変更内容
>>>>>>> main
```

意味:

- `<<<<<<< HEAD` から `=======` までが、自分のブランチ側の変更
- `=======` から `>>>>>>> main` までが、取り込んだ `main` 側の変更

残したい内容を選んで、次の記号を削除します。

```text
<<<<<<< HEAD
=======
>>>>>>> main
```

## 解消手順

```bash
git checkout コンフリクトしたブランチ名
git merge main
```

ファイルを修正します。

```bash
git add .
git commit -m "modify conflict"
git push origin コンフリクトしたブランチ名
```

## 練習用ファイル

`practice/conflict-message.txt` を複数人で編集すると、コンフリクトの練習ができます。

例:

1. Aさんが1行目を書き換えてPull Requestを出す
2. Bさんも同じ1行目を書き換えてPull Requestを出す
3. どちらかが先にmergeされる
4. もう一方のPull Requestでコンフリクトが起きる
